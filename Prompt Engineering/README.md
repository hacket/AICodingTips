## 🎭 角色扮演 Prompts

这些 Prompt 定义了特定的 AI 角色和行为模式。

### [RMS 灵感](role/RMS灵感.md)
**适用场景**: 需要从开源、自由软件的角度思考问题

模拟 Richard Stallman 的思维方式，强调:
- 自由软件理念

**何时使用**:
- 讨论软件许可和开源策略
- 评估技术方案的自由度
- 分析软件的伦理影响

## 💡 使用建议

### 如何选择合适的 Prompt

1. **明确你的目标**
   - 需要角色扮演? → 查看 `role/`
   - 需要深度思考? → 查看 `plan/`
   - 编程相关? → 查看 `开发/`

2. **组合使用**
   - 可以将多个 Prompt 组合使用
   - 例如: 用"思想催化剂"分析问题 + "代码架构"设计方案

3. **根据场景调整**
   - 这些 Prompt 都是基础框架
   - 可以根据具体需求进行调整和定制

### 最佳实践

**DO ✅**
- 先阅读 Prompt 全文，理解其设计思路
- 根据实际需求调整 Prompt 细节
- 测试效果，迭代优化

**DON'T ❌**
- 不要不加理解直接照搬
- 不要忽略 Prompt 的适用场景
- 不要期望一个 Prompt 解决所有问题

## 🔧 自定义 Prompt

想创建自己的 Prompt? 可以参考以下结构:

```markdown
# [Prompt 名称]

## 角色定位
清晰定义 AI 的角色和专长

## 核心能力
列出该角色的主要能力

## 工作方式
说明如何与用户交互、如何思考问题

## 约束条件
明确什么该做、什么不该做

## 示例对话
提供几个使用示例
```
